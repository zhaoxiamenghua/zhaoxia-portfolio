AI 协作作品集生产系统｜这个网站是如何被整理、生成和迭代出来的

这个页面放在作品集最后，用来复盘本网站本身的生产过程。

我用 Obsidian 整理长期材料和项目笔记，用 ChatGPT 梳理结构、重写文案、判断项目关系，用 Codex 协助网页代码生成与修改，用 GitHub 管理文件、版本和网页部署。

WORKFLOW

从材料到网页的协作流程

  1. 01长期材料整理：在 Obsidian 和文档中整理思感集、项目经历、图像材料和求职方向。
  2. 02结构判断：用 ChatGPT 反复讨论网站定位、项目顺序、主次关系和叙述语气。
  3. 03页面生成：将确定后的结构整理成 Codex 指令，生成或修改 HTML / CSS 页面。
  4. 04版本管理：通过 GitHub 保存代码、检查文件路径、管理页面版本和部署。
  5. 05人工复核：重新阅读网页，发现文案过度讨好、项目残留、链接缺失等问题，再继续修改。

TOOL ROLES

四个工具分别承担什么角色

OBSIDIAN

材料收集与长期记录

整理思感集、项目笔记、网页结构草稿、求职信息和灵感片段。它更像我的个人资料库。

CHATGPT

结构讨论与文案判断

帮助我梳理作品集主线、项目合并方式、页面文案语气和 Codex 修改指令。

CODEX

网页代码生成与修改

根据明确指令生成 HTML / CSS 页面，调整项目结构、链接、锚点和页面内容。

GITHUB

版本管理与网页部署

管理作品集仓库、页面文件、提交记录和 GitHub Pages 部署，也用于检查网页是否更新成功。

ITERATION LOG

这次网站经历过的关键修改

  1. 01从“做一个作品集网站”开始，先确认它不是普通视觉展示页，而是一次面向内容岗位的完整表达。
  2. 02结合网易 AI 向内容运营实习岗位，重新判断网站重点：不是只展示视觉能力，而是展示写作、内容判断、AI 工具使用和项目复盘。
  3. 03从最初的作品堆叠，调整为“一个人的内容生成系统”：长期记录、内容判断、AI 协作、项目输出和复盘。
  4. 04确定视觉风格不走明亮商业模板，而是使用低饱和、纸质、档案感、扫描书页和旧杂志式排版，让网站更接近个人材料档案。
  5. 05用 GitHub Pages 作为公开发布方式，用 VS Code / Codex / GitHub 协作推进网页搭建。
  6. 06最初曾尝试把网站写成“岗位匹配说明”，后来发现语气过于讨好 HR，于是改成“我平时如何做内容”。
  7. 07删除或弱化“证明、对应岗位、能力矩阵、候选人、匹配”等表达，改为“材料从哪里来、我如何判断、后来变成了什么”。
  8. 08重新分析《思感集》，确认它不是普通随笔栏目，而是整个作品集的底层材料系统。
  9. 09把《思感集》从“一个项目”提升为全站阅读入口，说明公众号文章、《坠落静默》、被单人、IMXS 和网站结构都与它有关。
  10. 10为《思感集》补充反复出现的主题：边界、控制、被看见、身体感受、爱与关系、自我接纳、表达与沉默、规训和人与世界的连接。
  11. 11把三篇公众号文章从“样本卡片”改成真实内容证据，并加入微信公众号阅读全文链接。
  12. 12为《我绝不认输》《总有人要仰望星空》《比铁手铐更坚固的，是纸手铐》分别保留站内文章拆解和站外原文入口。
  13. 13重新整理《坠落静默》的位置：它不是 AI 替我写出的剧本，而是来自长期心理感受和《思感集》材料的原创叙事。
  14. 14明确《坠落静默》中 AI 的作用：扩写、润色、镜头整理、提示词辅助；故事方向、情绪边界和最终取舍由我判断。
  15. 15将 IMXS、《坠落静默》和被单人合并为“AI 生成内容专题”，不再分散展示。
  16. 16重新定义 AI 生成内容专题的三条路径：IMXS 是品牌调性驱动，《坠落静默》是心理叙事驱动，被单人是产品设计和情绪 IP 驱动。
  17. 17发现被单人在观察档案中仍有残留，于是决定将它彻底移出 archive，只保留在 AI 生成内容专题中。
  18. 18将观察档案重新定位为“项目开始之前的材料来源”，只保留朋友圈观察、城市经验、用户体验、心理/艺术资料和产品设计资料。
  19. 19原本准备加入“AI 行业内容选题与样稿”，后来判断它不是目前已经真实持续完成的内容，于是删除这个主项目。
  20. 20把原来的 AI 样稿页改为最后栏目：“AI 协作作品集生产系统”，用来复盘这个网站本身如何被整理、生成和迭代。
  21. 21调整首页项目顺序：思感集 → AI 生成内容专题 → 展览体验 → 观察档案 → AI 协作作品集生产系统。
  22. 22将“AI 协作作品集生产系统”放在最后，而不是前面，避免它抢占真实内容项目的位置。
  23. 23补充项目内跳转：文章可以跳到原文和拆解，AI 生成内容专题可以跳到 IMXS / 坠落静默 / 被单人。
  24. 24补充项目页之间的上一页 / 下一页导航，让读者可以按新的阅读顺序浏览整个网站。
  25. 25尝试通过 ChatGPT 直接修改 GitHub 文件，但遇到写入权限限制，因此改为整理 Codex 指令和手动修改路径。
  26. 26在每轮修改后重新阅读网页，继续发现问题：文案是否过度包装、链接是否缺失、项目是否真实、栏目位置是否合理。
  27. 27统一全站 AI 协作说明，明确 ChatGPT、Codex 等工具参与结构梳理、文字修改和代码实现，但最终判断由我完成。
  28. 28最终形成当前版本：前面展示真实内容和项目，最后用一个 meta case 说明这个网站如何被我使用 AI 工具协作完成。

VERSION MAP

这个网站的版本演化路径

V0

目标确认：从求职需求开始

最开始的任务是为实习投递准备作品集，但很快发现不能只做一个“好看的网站”，而要说明我如何写作、观察、使用 AI 和组织项目。

V1

视觉方向：从模板感改为档案感

网站视觉从普通作品集模板转向低饱和、纸质、扫描书页和档案排版。这个风格更适合呈现长期材料、文字和项目痕迹。

V2

结构初版：项目开始被放到一起

思感集、AI 内容、AI 影像、展览和观察档案被放入同一个网站，但最初仍然更像多个项目并列陈列。

V3

岗位化过度：文案开始像能力证明

为了靠近 AI 内容运营岗位，页面一度出现较多“证明、匹配、能力矩阵”等表达。后来判断这种语气太用力，需要降调。

V4

底层系统：思感集被重新定位

经过重新整理，《思感集》不再只是一个写作项目，而成为全站材料来源：公众号文章、原创叙事、情绪 IP 和网站文案都能从这里找到起点。

V5

专题重组：AI 生成内容被合并

IMXS、《坠落静默》和被单人被合并成一个 AI 生成内容专题，分别对应品牌传播、原创叙事和产品设计。

V6

真实性检查：删除 AI 样稿栏目

由于 AI 行业样稿不是目前已经真实持续完成的内容，所以不再作为主项目展示，避免作品集显得临时包装。

V7

最后闭环：加入 AI 协作生产系统

原 AI 样稿页被改为最后栏目，用来说明整个网站如何通过 ChatGPT、Codex、GitHub 和 Obsidian 协作生产出来。

PROBLEM / DECISION / CHANGE

每一轮迭代解决了什么问题

发现的问题 我的判断 修改方式
网站最初容易像作品陈列 需要展示的不只是作品结果，而是内容如何从材料中长出来 加入“内容生成系统”的主线，把长期记录、判断和项目输出连接起来
文案太像在迎合岗位 不能写成“我证明我适合”，而要写成“我平时就是这样做内容” 删除“岗位匹配、能力矩阵、候选人”等表达，改成更平实的项目说明
思感集被当成普通写作项目 它其实是整个网站的材料底层 把思感集提升为全站阅读入口，并说明它如何进入其他项目
《坠落静默》容易被误解为 AI 写作项目 它的核心来自长期心理感受和叙事判断，AI 只是辅助工具 补充它与思感集的关系，明确 AI 没有决定故事方向
被单人分散在多个页面 它应该作为 AI 生成内容专题中的产品设计案例，而不是观察档案里的独立残留 从 archive 删除被单人，只保留在 AI 生成内容专题
AI 样稿栏目不够真实 不应该展示尚未持续完成的虚拟样稿 删除 AI 样稿主项目，改为真实发生过的网站 AI 协作生产案例
文章卡片无法阅读全文 作品集需要让读者继续进入真实内容，而不是只看摘要 接入三篇微信公众号原文链接，同时保留站内文章拆解
项目之间跳转不够清楚 读者需要按新的项目顺序连续阅读 调整首页项目顺序和每个项目页的上一页 / 下一页导航
AI 协作容易被写成炫技 最后栏目应该是制作说明，不是工具展示 把 ChatGPT、Codex、GitHub、Obsidian 的作用写成具体分工和边界
AI 参与程度需要透明 应该说明 AI 参与了整理和生成，但最终判断由我负责 全站保留克制的 AI 协作说明，并在最后栏目详细复盘

NOTE

项目小结

这个栏目放在最后，是因为它不是作品集的主角，而是这个网站的制作说明。

AI 对我最有价值的地方不是替我完成作品，而是帮我更快地整理材料、看见结构问题、生成可修改的草稿，并把想法推进到可发布的页面。

但它并不替我决定方向。哪些内容要保留，哪些项目要合并，哪些文案显得过度包装，哪些表达更接近我自己，这些仍然需要我反复阅读和判断。
