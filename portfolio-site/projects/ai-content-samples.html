PROJECT 05 / META CASE / AI WORKFLOW

AI 协作作品集生产系统｜这个网站是如何被整理、生成和迭代出来的

这个页面是全站最后的制作后记。

它记录我如何把思感集、公众号文章、AI 影像项目、展览经历、观察档案和求职方向重新放到同一个网站里。

ChatGPT、Codex、GitHub 和 Obsidian 参与了整理、生成、修改和发布；材料取舍、结构判断和最终表达由我完成。

SOURCE MATERIALS

网站搭建之前，材料并没有天然形成清楚的项目结构。第一步不是生成页面，而是确认每类材料在网站中承担什么作用。

01 / WRITING

思感集

长期写作、心理观察、关系经验和社会议题记录，构成网站的观点来源与语言基础。

02 / ARTICLES

公众号文章

三篇已发布长文保留原文链接、站内拆解和阅读数据，用来说明真实发布后的反馈。

03 / AIGC

AI 生成内容项目

IMXS、《坠落静默》和被单人被合并到同一专题，分别对应品牌、叙事和产品/IP 方向。

04 / FIELD NOTE

展览与观察档案

生命展、《越过珊丘》、城市经验、用户观察和资料库，作为项目开始之前的材料来源。

WORKFLOW

从材料到网页的协作流程

  1. 01长期材料整理：在 Obsidian 和本地文件中确认文章、项目、图片、研究资料和简历信息分别来自哪里。
  2. 02项目取舍：区分持续完成的项目、辅助材料和仍停留在设想阶段的内容。
  3. 03结构讨论：判断《思感集》、AI 生成内容、展览体验和观察档案之间的关系，重新安排阅读顺序。
  4. 04页面生成：把确认后的结构写成修改指令，由 Codex 协助生成或调整 HTML / CSS、链接和页面导航。
  5. 05版本管理：通过 GitHub 保存每轮修改并部署到 GitHub Pages，让页面进入真实公开环境。
  6. 06人工复核：在本地和线上重新检查语气、结构、链接、残留内容与响应式显示。

PROBLEM / DECISION / CHANGE

几次关键判断

发现的问题 我的判断 修改方式
网站最初容易像作品陈列 需要展示的不只是作品结果，而是内容如何从材料中长出来 加入“内容生成系统”的主线，把长期记录、判断和项目输出连接起来
文案太像在迎合岗位 不能写成“我证明我适合”，而要写成“我平时就是这样做内容” 删除“岗位匹配、能力矩阵、候选人”等表达，改成更平实的项目说明
思感集被当成普通写作项目 它其实是整个网站的材料底层 把思感集提升为全站阅读入口，并说明它如何进入其他项目
《坠落静默》容易被误解为 AI 写作项目 它的核心来自长期心理感受和叙事判断，AI 只是辅助工具 补充它与思感集的关系，明确 AI 没有决定故事方向
设想栏目缺少持续完成的内容 不应该把尚未发生的内容包装为主项目 删除设想栏目，改为真实发生过的网站 AI 协作生产案例
AI 协作容易被写成炫技 最后栏目应该是制作说明，不是工具展示 把 ChatGPT、Codex、GitHub、Obsidian 的作用写成具体分工和边界

APPENDIX

修改附录

下面保留较完整的修改轨迹。它不是主阅读内容，更像这次网站制作的边注。

  1. 01从“做一个作品集网站”开始，先确认它不是普通视觉展示页，而是一次面向内容岗位的完整表达。
  2. 02结合网易 AI 向内容运营实习岗位，重新判断网站重点：不是只展示视觉能力，而是展示写作、内容判断、AI 工具使用和项目复盘。
  3. 03从最初的作品堆叠，调整为“一个人的内容生成系统”：长期记录、内容判断、AI 协作、项目输出和复盘。
  4. 04确定视觉风格不走明亮商业模板，而是使用低饱和、纸质、档案感、扫描书页和旧杂志式排版，让网站更接近个人材料档案。
  5. 05用 GitHub Pages 作为公开发布方式，用 VS Code / Codex / GitHub 协作推进网页搭建。
  6. 06最初曾尝试把网站写成“岗位匹配说明”，后来发现语气过于讨好 HR，于是改成“我平时如何做内容”。
  7. 07删除或弱化“证明、对应岗位、能力矩阵、候选人、匹配”等表达，改为“材料从哪里来、我如何判断、后来变成了什么”。
  8. 08重新分析《思感集》，确认它不是普通随笔栏目，而是整个作品集的底层材料系统。
  9. 09把《思感集》从“一个项目”提升为全站阅读入口，说明公众号文章、《坠落静默》、被单人、IMXS 和网站结构都与它有关。
  10. 10为《思感集》补充反复出现的主题：边界、控制、被看见、身体感受、爱与关系、自我接纳、表达与沉默、规训和人与世界的连接。
  11. 11把三篇公众号文章从“样本卡片”改成真实内容证据，并加入微信公众号阅读全文链接。
  12. 12为《我绝不认输》《总有人要仰望星空》《比铁手铐更坚固的，是纸手铐》分别保留站内文章拆解和站外原文入口。
  13. 13重新整理《坠落静默》的位置：它不是 AI 替我写出的剧本，而是来自长期心理感受和《思感集》材料的原创叙事。
  14. 14明确《坠落静默》中 AI 的作用：扩写、润色、镜头整理、提示词辅助；故事方向、情绪边界和最终取舍由我判断。
  15. 15将 IMXS、《坠落静默》和被单人合并为“AI 生成内容专题”，不再分散展示。
  16. 16重新定义 AI 生成内容专题的三条路径：IMXS 是品牌调性驱动，《坠落静默》是心理叙事驱动，被单人是产品设计和情绪 IP 驱动。
  17. 17发现被单人在观察档案中仍有残留，于是决定将它彻底移出 archive，只保留在 AI 生成内容专题中。
  18. 18将观察档案重新定位为“项目开始之前的材料来源”，只保留朋友圈观察、城市经验、用户体验、心理/艺术资料和产品设计资料。
  19. 19原本准备加入一个面向行业选题的设想栏目，后来判断它不是目前已经真实持续完成的内容，于是删除这个主项目。
  20. 20把空出的页面改为最后栏目：“AI 协作作品集生产系统”，用来复盘这个网站本身如何被整理、生成和迭代。
  21. 21调整首页项目顺序：思感集 → AI 生成内容专题 → 展览体验 → 观察档案 → AI 协作作品集生产系统。
  22. 22将“AI 协作作品集生产系统”放在最后，而不是前面，避免它抢占真实内容项目的位置。
  23. 23补充项目内跳转：文章可以跳到原文和拆解，AI 生成内容专题可以跳到 IMXS / 坠落静默 / 被单人。
  24. 24补充项目页之间的上一页 / 下一页导航，让读者可以按新的阅读顺序浏览整个网站。
  25. 25尝试通过 ChatGPT 直接修改 GitHub 文件，但遇到写入权限限制，因此改为整理 Codex 指令和手动修改路径。
  26. 26在每轮修改后重新阅读网页，继续发现问题：文案是否过度包装、链接是否缺失、项目是否真实、栏目位置是否合理。
  27. 27统一全站 AI 协作说明，明确 ChatGPT、Codex 等工具参与结构梳理、文字修改和代码实现，但最终判断由我完成。
  28. 28最终形成当前版本：前面展示真实内容和项目，最后用一个 meta case 说明这个网站如何被我使用 AI 工具协作完成。
